Wayra Web Backend is an Express.js REST API for restaurant operations. This guide walks you through cloning the project, connecting to MySQL, starting the server, and verifying everything works with a real login request.

## Prerequisites

- Node.js 18 or later
- A MySQL database (local or hosted) with the Wayra schema applied
- npm (comes with Node.js)

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/carrioneilnacr-art/Wayra_Web_Backend.git
    cd Wayra_Web_Backend
    ```
  </Step>
  <Step title="Install dependencies">
    <CodeGroup>
    ```bash npm
    npm install
    ```
    ```bash yarn
    yarn install
    ```
    </CodeGroup>

    The project depends on `express`, `cors`, `dotenv`, and `mysql2`.
  </Step>
  <Step title="Configure environment variables">
    Create a `.env` file in the project root:

    ```bash .env
    DATABASE_URL=mysql://user:password@localhost:3306/wayra_db
    PORT=3000
    ```

    See [Configuration](/configuration) for all available options.
  </Step>
  <Step title="Start the server">
    <CodeGroup>
    ```bash Development
    npm run dev
    ```
    ```bash Production
    npm start
    ```
    </CodeGroup>

    When the server starts you should see:
    ```
    🚀 SERVIDOR WAYRA NIKKEI ACTIVO EN PUERTO 3000
    ```
  </Step>
</Steps>

## Verify with your first request

Test authentication by logging in with a valid user:

```bash
curl -X POST http://localhost:3000/api/login \
  -H "Content-Type: application/json" \
  -d '{"user": "admin", "pass": "your_password"}'
```

A successful response returns the user record:

```json
{
  "success": true,
  "usuario": {
    "id_usuario": 1,
    "nombre": "Admin",
    "rol": "admin"
  }
}
```

<Tip>
  Use `npm run dev` during development — it uses nodemon to auto-restart the server on file changes.
</Tip>
